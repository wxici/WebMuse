# Manual PoC History

Before WebMuse became a public OSS workbench, the workflow was manually tested through a reference-style website reconstruction process.

This proof of concept was about validating a workflow, not copying a third-party website.

## Early manual loop

The early manual workflow included:

1. Observing a reference website.
2. Collecting screenshots, recordings, or visual notes.
3. Generating an initial AI/Codex-assisted page.
4. Comparing rhythm, whitespace, typography scale, image mood, transition behavior, and visual hierarchy.
5. Applying manual CSS and visual tuning.
6. Validating the improved result.
7. Identifying which steps should become repeatable software features.

## What the manual loop showed

The manual process showed that AI-generated first drafts can capture broad structure and mood, but they are rarely enough for final delivery.

Common gaps included:

- spacing accuracy;
- motion direction;
- mask or hover behavior;
- image brightness;
- dark/light transition mood;
- information density;
- brand replacement;
- section rhythm;
- final visual polish.

## Why this matters

The proof of concept helped define WebMuse as a workflow tool, not a one-shot prompt tool.

The useful software features identified from the manual loop include:

- observation packages;
- construction packages;
- tuning overrides;
- validation reports;
- sandbox boundaries;
- dry-run plans;
- proof-check artifacts;
- approval gates;
- rollback readiness;
- manual fallback.

## Role of recording and frame extraction

Recording and frame extraction remain useful for correcting complex visual details that are hard to capture from static code alone.

Examples include:

- hover behavior;
- scroll-triggered effects;
- mask direction;
- crescent or shape transitions;
- light/dark image transitions;
- timing and easing;
- before/after visual comparisons.

## Boundary

This PoC history does not authorize copying third-party brand assets, logos, proprietary images, or full website identities.

The goal is reference-style reconstruction:

```text
observe -> understand -> transform -> create a user-owned branded result
```
