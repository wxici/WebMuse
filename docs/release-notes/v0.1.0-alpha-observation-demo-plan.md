# v0.1.0-alpha observation demo plan

This note records the alpha observation demo decision for the Manual PoC 001 OSS presentation update.

## What exists

The repository currently contains historical WebRebuildRecorder application source and foundation services for project manifests, sandbox validation, readiness checks, dry-run records, approval gates, package preparation, and foundation self-tests.

The public documentation also describes existing or historical support areas such as source-site observation, recording, frame extraction, and observation-package preparation.

## What remains needed

Before a public binary observation demo is packaged, WebMuse still needs a deliberate release pass:

1. verify the WPF application startup manually on a clean Windows machine;
2. verify recording and frame extraction behavior with safe local sample material;
3. verify FFmpeg/FFprobe availability and fallback messaging;
4. verify observation package generation end to end;
5. add a release README with limitations and safety notes;
6. confirm no private recordings, extracted frames, customer materials, credentials, local paths, or generated output-site artifacts are included.

## Demo package decision

No binary package is produced in this round.

The current task updated public Manual PoC evidence and repository presentation. It did not manually validate the WPF recording, frame extraction, or observation package workflows end to end, so publishing `WebMuse-v0.1.0-alpha-observation-demo.zip` would overstate readiness.

## Next step

Prepare the alpha observation demo as a dedicated release-readiness task after a clean Windows manual run and package audit.
