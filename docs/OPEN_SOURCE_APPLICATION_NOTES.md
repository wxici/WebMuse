# Open Source Application Notes

This document prepares short, accurate descriptions for the Codex for OSS application.

## Repository qualification summary

Short version under 500 characters:

```text
WebMuse is an early-alpha open-source Windows desktop workbench for safer Codex-assisted reference-style website reconstruction workflows. It grew from a manually validated workflow and focuses on observation packages, construction tasks, sandbox validation, dry-run checks, approval gates, rollback readiness, and manual fallbacks before real AI execution.
```

## Maintainer role

The repository is maintained by the project creator as the primary maintainer.

The maintainer is responsible for:

- project direction;
- roadmap planning;
- issue triage;
- safety boundaries;
- release preparation;
- documentation;
- Codex task design;
- review of AI-assisted changes.

## Current limitations

WebMuse is early alpha.

Not currently enabled:

- real Codex CLI execution;
- OpenAI API calls;
- website generation;
- WebView2 preview;
- tuning panel;
- public release installer.

The project currently emphasizes safety foundations and public OSS readiness.
